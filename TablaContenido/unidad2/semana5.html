Unidad 2

Semana 13

Temas:

  • Desarrollo de API Rest con Spring
  • HTTP (Protocolo de Transferencia de Hipertexto)
  • MVC (Modelo-Vista-Controlador)
  • Arquitectura de API Rest

Definiciones:

1. API Rest: Una interfaz de programación de aplicaciones que sigue los principios de arquitectura REST para facilitar la comunicación entre sistemas. En el contexto de Spring, esto implica el uso de herramientas y convenciones específicas para desarrollar servicios web RESTful.
2. HTTP: Protocolo fundamental para la comunicación en la web, donde se define cómo los mensajes deben ser transmitidos y cómo los navegadores y servidores deben responder a diversos comandos.
3. MVC (Modelo-Vista-Controlador): Un patrón de diseño que separa la lógica de una aplicación en tres componentes interconectados: Modelo (gestión de datos), Vista (presentación de la interfaz de usuario) y Controlador (manejo de la lógica de negocio y la interacción entre Modelo y Vista).
4. Arquitectura de API Rest: Un conjunto de principios y restricciones que guían el diseño de servicios web para ser eficientes, escalables y fáciles de entender. Esto incluye la utilización de HTTP como protocolo de comunicación y la representación uniforme de recursos.

Procedimiento:

° Desarrollo de API Rest con Spring: Utilizar el marco de trabajo Spring para crear controladores, definir rutas y gestionar recursos, siguiendo las mejores prácticas de desarrollo RESTful.
° HTTP: Comprender los métodos HTTP (GET, POST, PUT, DELETE) y sus códigos de estado, implementando correctamente las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en la API.
° MVC: Organizar el código en capas separadas para garantizar una gestión clara y eficiente de la lógica de negocio, la presentación y la manipulación de datos.
° Arquitectura de API Rest: Aplicar los principios de REST, como la representación uniforme de recursos, la navegabilidad y la independencia de estado, para diseñar una API que sea coherente y fácil de consumir.

Resultado:

° Laboratorio 13: Desarrollo de una API Rest y Despliegue
° Al completar este laboratorio, se alcanzará un entendimiento profundo de cómo desarrollar una API Rest utilizando Spring, aplicando los conceptos de HTTP, MVC y la arquitectura REST. Además, se adquirirán habilidades para desplegar y gestionar eficientemente la API en entornos de producción.
